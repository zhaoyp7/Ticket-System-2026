# priority_queue

## 实现细节

你需要完成的内容：

- 构造（无参，拷贝）
- 析构
- 赋值（拷贝）
- 插入元素 push(const T&) -> void（对数复杂度）
- 获取现存元素最大值 top() const -> const T&（常数复杂度）
- 删除现存元素最大值 pop() -> void（对数复杂度）
- 获取现存元素数量 size() const -> size_t（常数复杂度）
- 检测是否可取元素 empty() const -> bool（常数复杂度）
- 清空队列 clear() -> void（线性复杂度）
- 合并两个优先队列 merge(sjtu::priority_queue& other) → void（对数复杂度）

具体细节可以查看下发的 `priority_queue.hpp` 框架。最终也只需要提交该文件内容。

# 实现要求

- 你能使用的头文件仅限于下发框架中提供的头文件；
- 注意内存安全，有内存泄漏测试；
- 你的实现需要保证，若成员函数（push, pop, top, merge）需要传递异常，则其应当在不影响优先队列状态（对于 merge 函数，另一个优先队列的状态也不能被影响）的前提下完成。“不影响”指的是优先队列的状态应当回退到该操作调用前，后续的操作表现应当与它并未被该异常操作调用的情况一致。具体请参看测试数据；
- 异常传递时传递原异常，对其不做任何修饰处理。

- 完成一份复杂度分析报告。复杂度分析报告的要求如下：

  - 可能需要定义辅助函数/性质并分析其复杂度，可能需要摊还分析/势能分析等高级方法。
